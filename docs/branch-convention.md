# 브랜치 네이밍 규칙

## 📌 브랜치 전략

| 유형       | 접두어       | 설명                                |
|------------|--------------|-------------------------------------|
| 기능 개발  | `feature/`   | 새로운 기능 개발 시                 |
| 버그 수정  | `fix/`       | 버그나 에러 수정 시                |
| 리팩토링   | `refactor/`  | 코드 구조 개선 (기능 변경 없음)    |
...

## 📍 브랜치 예시

- `feature/login-api`
- `fix/null-pointer-exception`
...
